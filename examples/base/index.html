Основная документация по движку

Быстрый старт

Минимальное требование для запуска движка — любой HTTP сервер, кладёте исходники движка в корневую папку сервера, и движек готов для работы.

Для плагинов search, globalNav и bubble, которые включены в ядро, но по умолчанию выключены, требуется PHP 5.3 на сервере и установка прав на запись в папку /data.

Структура

Основная структура движка основана на модулях и плагинах, есть корневые модули, что находятся в папке /core/js/modules, есть отключаемые плагины в /plugins.

Для работы с движком, нет необходимости править файлы в папке/core, для этого есть папка с пользовательскими переопределениями настроек и шаблонов /user.

Папка /data предназначена для пользовательских данных и герерируемых файлов.

Плагины

Часть плагинов включена в движок по умолчанию в выключенном состоянии, что бы изменть настройки плагинов, нужно поправить файл пользовательских настроек /user/js/options.js.

pluginsEnabled : { //Plugins enabling //bubble: true, //debugmode: true //globalNav: true, //search: true }

Что бы активировать плагин, достаточно раскомментировать параметр с его названием.

Плагины навигации и поиска

search, globalNav

Плагины поиска и навигации по документациям работают на основе геренируемого JSON файла, который создаются с помощью скрипта/плагина /file_tree_generator/core/file_tree_generator.php. Что бы скрипт смог сложить JSON в нужную папку, нужно обязательно уставить права записи на папу /data.

Что бы автоматизировать генерацию JSON, советуем настроить cron таск на запуск скрипта, или запускать скрипт вручную.

Организация документаций

Документация проекта расположена в папке examples, для того, что бы сделать подобные страницы для своих нужнд, достаточно повторить структуру раздела Документация (/examples).
