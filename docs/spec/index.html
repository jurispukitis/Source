Spec page markup

Documentation text

Documentation text in sections is marked up by simple HTML, using tags: p, ul, li, h1, h2, h3, h4. Only those tags are decorated and have style reset from project CSS.

In nearest updates we will add Markdown support for easy text formatting inside documentation sections.

Starting template will always be available for fast new spec page creation, just copy it to another folder.

Metas

To add some machine readable information for plugins like Global Navigation and Search we use metas:

<meta name="keywords" content="create spec, spec markup"> <meta name="author" content="Robert Haritonov">

Also, using meta information we're separating page roles like navigation:

<meta name="source-page-role" content="navigation">
